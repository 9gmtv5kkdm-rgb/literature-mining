## 外部依赖与降级声明

- **外部依赖**：联网检索（`web_search`）用于设计检索策略、核实 PRISMA 最新标准；并可路由至外部检索类技能（如 `literature-review` / `literature-reviewer-skill`，外部检索技能，需另装）执行实际文献检索。
- **降级策略**：本技能核心为“读文献”的方法论工具包，不依赖外部检索即可运行；若联网未配置或外部检索技能不可用，明确提示用户并基于其已提供的文献继续分析，不伪造文献、不静默失败。
- **权限**：所有外部调用均经用户授权；未配置时不阻断核心阅读笔记/矩阵/Gap 分析能力。

本技能提供文献阅读方法论工具，**严禁用于以下行为**：

1. **禁止编造虚假文献**：不得利用本技能生成不存在的论文标题、作者、期刊、DOI 或任何形式的虚假参考文献。所有文献必须来自真实检索（Web of Science、PubMed、Scopus、CNKI 等可信数据库）。

2. **禁止生成虚假文献综述**：文献矩阵和 Gap 识别仅限对用户提供的真实文献进行分析，不得凭空构造"文献综述"来填充论文章节。所有分析输出须标注检索来源与日期。

> **AI 声明**：本输出由 AI 辅助生成，请核对原始来源后使用。用户应独立核实每篇文献的真实性与相关性。

# 文献挖掘与阅读笔记 (Literature Mining & Reading Notes)

## 定位与关系

### 本技能是什么

`literature-mining` 是**"阅读文献的方法论工具包"**。它回答的问题是：

> *"我已经找到了一批论文——现在怎么高效地从中提取信息、构建知识体系、找到研究空白？"*

### 与已有技能的关系

SkillHub 上已有多个优秀的文献检索技能（均为外部检索技能，需另装）：

| 技能 | 核心功能 |
|------|----------|
| `literature-review` v1.2.0 | 检索 Semantic Scholar / OpenAlex / Crossref / PubMed |
| `literature-reviewer-skill` v3.0 | 检索 CNKI / WoS / ScienceDirect，8阶段工作流 |
| `literature-search` v1.0.3 | 检索 8 大数据库 |
| `literature-manager` v1.1.1 | 搜 + 下 + 转 + 整 + 审全套管理 |

**这些技能负责"找文献"——本技能负责"读文献"。**

它们是互补关系：先用 `literature-review` 或 `literature-reviewer-skill`（外部检索技能，需另装）检索到目标文献集合，再用本技能提供的工具包进行系统化精读和分析。

> 本套件内的本地检索技能 `academic-search-pro` 同样负责"找文献"（初检/广搜），初检/广搜见 academic-search-pro（兄弟技能）。
> 与 academic-search-pro 互补：本技能管精读管理，初检广搜走学术检索（academic-search-pro）。

```
检索技能 (找到文献)          本技能 (读懂文献)
──────────────────────       ──────────────────────
literature-review            → 结构化阅读笔记
literature-reviewer-skill    → 文献矩阵构建
literature-search            → Research Gap 识别
literature-manager           → 证据质量评价
                              → PRISMA 流程图
                              → 检索策略设计
```

## 推荐工作流

1. **检索策略设计** → 阅读 `references/search-strategy.md`，用 PICO 框架明确检索范围
2. **文献检索** → 使用 `literature-review` / `literature-reviewer-skill`（外部检索技能，需另装）执行检索
3. **PRISMA 筛选记录** → 阅读 `references/prisma-guide.md`，记录筛选过程并生成流程图
4. **结构化阅读** → 阅读 `references/reading-notes.md`，对核心文献深度阅读，对次要文献快速浏览
5. **文献矩阵构建** → 阅读 `references/literature-matrix.md`，横向比较所有纳入文献
6. **Gap 识别** → 阅读 `references/gap-identification.md`，从矩阵中系统性发现研究空白
7. **证据质量评价** → 阅读 `references/evidence-hierarchy.md`，对纳入文献进行证据分级

## 场景路由

| 场景 | 目标 | 输入 | 输出 | 红线 |

| 文献精读 | 对核心文献深度阅读做笔记 | 文献PDF+阅读目标 | 结构化笔记 | 非检索文献 |

| 研究空白识别 | 从文献矩阵中找到Gap | 文献矩阵 | Gap声明+创新点 | 矩阵完成后进行 |

决策树：
```
用户请求 → 场景匹配
   ├─ 命中1个场景 → 直接执行
   ├─ 命中多个场景 → 按优先级从高到低执行（如文献矩阵构建 > 研究空白识别 > 证据质量评价 > 文献精读）
   └─ 未命中 → 回退到用户指定模式
```


## 质量标准与红线

### 四维质量评分

| 维度 | 定义 | 优秀(3分) | 达标(2分) | 不及格(0分) |
|------|------|-----------|-----------|------------|
| 笔记结构化程度 | 阅读笔记是否覆盖关键要素 | 全要素覆盖 | 主要要素 | 碎片化 |
| 矩阵逻辑性 | 对比维度是否合理、可分析 | 一切多维可比 | 对比不全面 | 无逻辑结构 |
| Gap创新性 | 识别的研究空白是否有效 | 明确创新性 | 需要再界定 | 已有研究覆盖 |
| 证据分级正确性 | 分级方法是否科学 | 标准分级 | 略有偏差 | 分级错误 |

加权总分 = Σ(维度分) / (4×3) × 100
优质线：≥75分 ｜ 合格线：≥50分 ｜ 重做线：<50分

## 🔒 硬边界声明（不自动发布 / 不自动改写源文件 / 关键操作需用户确认）

- ❌ **不自动发布**：任何分析产出都不会被自动发布到任何平台，发布需你明确确认。
- ❌ **不自动改写源文件**：我不会擅自修改、覆盖或删除你本地的文献、笔记或正文文件；产出以"建议 + 文本"形式呈现，由你决定是否落盘。
- ❌ **关键操作需用户确认**：涉及覆盖原稿、替换内容、对外发送等动作，必须先征得你的确认。
- ❌ **不检索文献**：本技能只阅读分析你已提供的文献，检索请走检索类技能。

## 诊断系统

### 快车道：案例库

| 模式名 | 症状 | 修正操作 | 示例 |

| 笔记无焦点 | 读了整篇但不知道记什么 | 先回答"这篇解决了什么问题"再读 | 结构化笔记 |

## 联网规则

1. 当需要文献检索策略时，搜索用户需要设计检索策略时数据库语法
2. 当需要PRISMA最新标准时，搜索更新PRISMA流程图标准时确认
3. 任何联网获取的信息必须注明来源
4. 联网结果仅供辅助决策，不替代用户自身判断
5. 如果联网失败，基于已有知识和模板继续工作并告知用户
6. 超出知识范围时优先联网搜索而非猜测
