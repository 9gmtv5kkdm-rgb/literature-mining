# 文献矩阵方法论 / Literature Matrix Methodology

> 把 20-30 篇文献变成一张可分析的二维表——横向比较、模式发现、Gap 定位

---

## 什么是文献矩阵

文献矩阵（Literature Matrix）是将多篇文献的关键信息按照统一维度整理成二维表格的系统化方法。它不是简单的"文献列表"，而是一种**分析工具**——通过矩阵的横向和纵向扫描，你可以：

- **横向比较**：在同一维度下比较不同论文的处理方式
- **纵向看趋势**：按时间排列，发现研究焦点的演变
- **交叉定位空白**：找到"所有论文都没覆盖的维度组合"

---

## 矩阵维度设计

### 推荐 10 维度（可根据研究领域调整）

| # | 维度 | 说明 | 填写方式 |
|---|------|------|----------|
| 1 | 研究问题 | 论文试图回答的核心问题 | 一句话概括 |
| 2 | 方法类型 | 研究方法的类别 | 从分类词表中选取 |
| 3 | 数据来源 | 数据从哪里来 | 数据集名称/采集方式 |
| 4 | 样本量 | 研究对象规模 | N=? |
| 5 | 关键发现 | 最重要的实证结果/理论贡献 | 一句话 + 关键数字 |
| 6 | 创新点 | 论文的独特贡献 | 理论/方法/应用 |
| 7 | 局限性 | 作者自述或你发现的局限 | 要点列表 |
| 8 | 外部效度 | 结论能否推广到其他场景？ | 高/中/低 + 理由 |
| 9 | 与我研究的差距 | 离直接支撑你的研究还有多远？ | 近/中/远 + 说明 |
| 10 | 引用优先级 | 在论文中引用的重要程度 | A（核心引）/ B（建议引）/ C（可选） |

### 维度定制建议

不同研究领域可能需要不同维度。在基础 10 维之上，可根据需要增减：

| 领域 | 建议增加的维度 |
|------|---------------|
| 实验科学 | 实验条件、测量精度、效应量 |
| 社会科学 | 理论框架、样本特征（年龄/性别/文化）、测量工具 |
| 工程/CS | 代码可用性、硬件环境、推理延迟 |
| 医学/健康 | 随访时长、不良事件、CONSORT 评分 |

## 如何从矩阵中发现 Gap

### 三步法

```
横向扫描缺失值 → 纵向对比趋势 → 交叉定位空白区
```

#### 第一步：横向扫描缺失值

逐列检查矩阵，找出"该列大面积空白"或"该列选项单一"的情况。

```markdown
示例发现：
- "样本量"列：18/20 篇论文使用 N<1,000 的小型数据集 → 大样本研究缺乏
- "数据来源"列：15/20 篇使用公开 benchmark → 工业真实数据验证缺乏
- "外部效度"列：17/20 篇标注为"低" → 结论通用性存疑
```

#### 第二步：纵向对比趋势

将文献按时间排列（从早到晚），观察每个维度随时间的变化。

```markdown
示例发现：
- 2018-2020：方法类型→传统统计模型为主
- 2021-2023：方法类型→深度学习开始出现但集中在 CV/NLP 领域
- 2024-2025：方法类型→仍无基于 Transformer 的表格数据研究
  → Gap：Transformer 在表格数据中的应用是近年趋势，但尚不充分
```

#### 第三步：交叉定位空白区

找到"两列交叉处没有任何论文覆盖"的区域——这是最有力的 Gap 证据。

```markdown
示例发现：
- "方法=可解释AI" × "领域=工业制造" → 覆盖 0 篇
- "样本量>10,000" × "数据来源=真实工业数据" → 覆盖 0 篇
- "研究设计=纵向追踪" × "任务=预测性维护" → 覆盖 1 篇（仅案例报告）
```

## 完整矩阵模板

### 基础版（Markdown 表格）

适合 10-20 篇文献的规模。在 Markdown 编辑器中可直接维护。

```markdown

|---|----------|---------|------|---------|---|---------|--------|--------|---------|--------------|--------|
| 1 | ... | ... | ... | ... | ... | ... | ... | ... | ... | ... | A/B/C |
| 2 | ... | ... | ... | ... | ... | ... | ... | ... | ... | ... | A/B/C |
| ... | ... | ... | ... | ... | ... | ... | ... | ... | ... | ... | ... |

### Gap 总结

- **理论 Gap**：[列出]
- **方法 Gap**：[列出]
- **数据 Gap**：[列出]

```

## 使用流程

### 阶段 1：初筛（20-30 篇）

通过检索技能获得初始文献集后，快速浏览标题和摘要，将可能相关的 20-30 篇纳入矩阵候选。

### 阶段 2：快速填表（每篇 10-15 分钟）

对每篇论文快速阅读（摘要 + 引言 + 结论），填充矩阵的前 5 列（研究问题 / 方法 / 数据 / N / 关键发现）。

### 阶段 3：着色标记

使用颜色标记矩阵中的关键模式：

| 颜色 | 含义 | 示例 |
|------|------|------|
| 🔴 红 | 该方法/数据组合尚无文献覆盖 | "工业数据 + 可解释AI"列为空 |
| 🟡 黄 | 覆盖薄弱（仅 1-2 篇且质量低） | 仅 1 篇案例报告，证据等级低 |
| 🟢 绿 | 覆盖充分（≥3 篇高质量研究） | 多个 RCT 验证同一方法 |
| ⚪ 灰 | 与我的研究无关 | 纯粹理论性论文，无实证 |

### 阶段 4：Gap 总结

基于着色结果，写一段 Gap 总结（可参考 `gap-identification.md` 中更详细的框架）。

```markdown
### Gap 总结示例

基于对 22 篇文献的矩阵分析，识别以下研究空白：

1. **数据 Gap**：16/22 篇（73%）使用公开 benchmark 数据集，
   仅 1 篇使用真实工业数据，且该篇未公开数据集。
   → 缺乏工业制造场景的大规模、可重复验证数据集。

2. **方法 Gap**：所有深度学习相关论文（8 篇）集中在图像/文本模态，
   涉及结构化表格数据的论文全部使用传统方法（LR/XGBoost）。
   → 深度学习方法在工业表格数据上的适用性尚未被系统验证。

3. **应用 Gap**：仅 2 篇论文讨论了从实验环境到生产环境的部署挑战，
   且均为定性讨论，无定量实验。
   → 工业场景中的 MLOps/部署鲁棒性研究空白。

4. **理论 Gap**：大多数论文报道"我们的方法性能最好"但未解释"为什么好"——
   缺乏对设计选择的理论分析或消融实验。
   → 方法论背后的理论依据成疑。
```

## 4 篇虚构论文的矩阵示例

以下用 4 篇虚构论文展示矩阵的实际填充效果：

```markdown
## 文献矩阵：工业制造场景中的质量预测方法（日期：2025-06-22）

| # | 作者/年份 | 研究问题 | 方法 | 数据来源 | N | 关键发现 | 创新点 | 局限性 | 外部效度 | 与我研究的差距 | 优先级 |
|---|----------|---------|------|---------|---|---------|--------|--------|---------|--------------|--------|
| 1 | Doe et al., 2025 | 深度神经网络 vs 梯度提升树在表格数据的普适性比较 | 基准实验 | 45 个公开表格数据集 (OpenML/UCI) | 200–50,000 | 树模型（XGBoost/LightGBM）在中等数据集上仍是强基线；FT-Transformer 在 N>30,000 时才显优势 | 首次系统纳入训练成本作为评价维度 | 未覆盖工业制造数据（全部是通用数据集），未考虑特征工程 | 中（通用发现需工业场景验证） | 中 — 提供方法选择的一般指导，但场景不匹配 | A |
| 2 | Smith & Kumar, 2024 | 能否用振动传感器数据 + 轻量级 ML 预测 CNC 加工质量？ | 实验验证 | 自采集 CNC 加工中心传感器数据 | 1,200 个加工循环 | 随机森林 + 时域特征在 12 个质量指标上 MAE < 5% | 在真实工业车间验证了 ML 方案 | 仅限 CNC 加工场景，通用性存疑；N=1,200 较小 | 中（仅 CNC 场景） | 近 — 同为制造场景，但工艺不同 | B |
| 3 | Chen et al., 2023 | 迁移学习能否将源工艺的质量模型迁移到目标工艺？ | 实验验证 | 仿真数据 + 1 条真实产线 | 仿真 50,000 + 真实 800 | 领域对抗迁移在目标工艺上 F1 提升 12%，但需要至少 200 目标域标注样本 | 首次在制造领域验证迁移学习 | 仅测试了"冲压→锻造"迁移，未验证更多工艺组合；仿真数据过于理想化 | 低（仅一个工艺对） | 近 — 迁移学习思路可借鉴，但需验证更多工艺 | B |
| 4 | Williams & Zhao, 2022 | 工业质量预测综述：方法、挑战与未来方向 | 系统综述 | 2010-2021 年发表的 150 篇论文 | N/A | 70% 研究使用公开数据集；仅 8% 部署到真实产线；数据质量和标注一致性是最大瓶颈 | 提供全面的分类框架 | 仅覆盖到 2021 年；综述范围宽泛，缺乏定量 meta-analysis | 高（综述结论覆盖多场景） | 近 — 提供背景框架和引用坐标系 | A |

### 着色分析

| 维度 | 🔴 红（空白） | 🟡 黄（薄弱） | 🟢 绿（充分） |
|------|-------------|-------------|-------------|
| 方法 | — | 迁移学习（仅1篇，工艺有限） | 传统 ML（3篇） |
| 数据来源 | 真实工业大规模开放数据集（0篇） | 真实工业数据（2篇但 N 小） | 公开通用数据集（1篇代表） |
| 外部效度 | 高外部效度 + 工业场景（0篇） | — | 中低外部效度（4篇） |
| 应用 | 部署到生产环境的全过程研究（0篇） | 真实产线部署（1篇，部分） | 离线实验（4篇） |
```

## 工具支持

### 推荐工具

| 工具 | 适用规模 | 优势 |
|------|---------|------|
| Excel / Google Sheets | < 50 篇 | 颜色标记方便，筛选排序灵活 |
| Notion / Airtable | 50-200 篇 | 数据库视图，支持关联和标签 |
| Markdown 表格 | < 30 篇 | 纯文本，版本控制友好 |
| Python pandas DataFrame | 不限 | 可编程分析，自动统计 Gap |
| NVivo / ATLAS.ti | 不限 | 质性研究专用，编码功能强 |

### Excel 操作提示

```text
1. 表头冻结：View → Freeze Panes → Freeze Top Row
2. 条件格式：选择"与我研究的差距"列 → Conditional Formatting → "近"标绿，"中"标黄，"远"标红
3. 筛选：Data → Filter → 可在任意列快速筛选
4. 透视表：Insert → PivotTable → 可统计"方法类型 × 年份"交叉分布
```

## 维护建议

1. **矩阵是活的**：随着阅读深入不断更新（新的发现、新的局限认知等）
2. **版本管理**：每次重大更新后保存一个快照（命名为 `literature-matrix-v1-20250622.xlsx`）
3. **链接笔记**：矩阵中的每篇文献应链接到对应的深度阅读笔记文件
4. **定期回顾**：每月检查一次 Gap 总结是否仍然成立，新发表的论文是否已填补
